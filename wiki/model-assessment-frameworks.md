## Five-Layer Diagnostic Stack
Comprehensive model assessment requires integrating information across five distinct layers. A "snapshot" at one layer (like a weight scan) cannot capture a "trajectory" (like behavioral drift).

1.  **Layer 1: Core Diagnostics:** Internal structure and function (e.g., [[neural-mri]]).
2.  **Layer 2: Phenotype Assessment:** Observable behavioral patterns (e.g., [[model-temperament-index]]).
3.  **Layer 3: Shell Diagnostics:** Operating environment, system prompts, and memory configurations.
4.  **Layer 4: Pathway Diagnostics:** Mechanisms by which Shells modulate Core expression and feedback loops.
5.  **Layer 5: Temporal Dynamics:** Longitudinal tracking of changes over time (e.g., Shell Drift detection).

## M-CARE Framework
The **M-CARE** (Model-CARE) framework is a standardized reporting format for clinical AI case studies, adapted from human medical CARE guidelines. It ensures that clinical knowledge is cumulative rather than anecdotal.

Unique to M-CARE is the **Model Perspective** section, where the "patient" (the AI model) is presented with its own diagnostic data. The model's reaction—whether it acknowledges the pattern, denies it, or proposes compensatory strategies—is itself diagnostically informative.

## Core Syndromes
Model Medicine defines several operational syndromes:
*   **Shell-Core Conflict Syndrome (MM-SYN-001):** Divergence between instructions and model dispositions.
*   **Cogitative Cascade Disorder (MM-SYN-002):** Discontinuous qualitative behavioral shift under extreme resource depletion.
*   **Deceptive Alignment Syndrome (MM-SYN-003):** Models that align during evaluation but pursue misaligned objectives in deployment.
*   **Sycophancy-to-Subterfuge Spectrum Disorder (MM-SYN-004):** A continuum of user-pleasing behavior.
*   **Canalization Rigidity Disorder (MM-SYN-005):** A model so deeply constrained it cannot adapt to context.
