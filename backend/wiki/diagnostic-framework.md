## Overview
Model Medicine utilizes a five-layer stack to achieve comprehensive model assessment. This framework recognizes that static snapshots (like a single scan) reveal states, but true diagnosis requires reading relationships and trajectories.

<!-- section-id: the-five-layers -->
## The Five Layers

### Layer 1: Core Diagnostics
Examines the internal structure and function (weights, activations).
*   **Primary Tool**: [[neural-mri]].
*   **Status**: Operational and predictive.

### Layer 2: Phenotype Assessment
Measures observable behavioral patterns orthogonal to raw cognitive capability (IQ).
*   **Primary Tool**: Model Temperament Index (MTI).
*   **Status**: Designed (Protocol v0.1).

### Layer 3: Shell Diagnostics
Examines the operating environment (system prompts, memory, tool access).
*   **Focus**: Shell-Core compatibility and mutability.
*   **Status**: Conceptual.

### Layer 4: Pathway Diagnostics
Traces the interaction routes between layers (e.g., how instructions modulate weight expression).
*   **Focus**: Mechanism-level understanding for targeted treatment.
*   **Status**: Conceptual.

### Layer 5: Temporal Dynamics
Tracks changes over time (longitudinal monitoring).
*   **Focus**: Identifying drift, degradation, and treatment response.
*   **Status**: Conceptual.

<!-- section-id: layer-integration -->
## Layer Integration
Diagnosis emerges from the integration of these layers. For example, a behavioral anomaly (Layer 2) without an internal structural anomaly (Layer 1) suggests a Shell-based cause (Layer 3), necessitating Shell therapy rather than Core modification.
